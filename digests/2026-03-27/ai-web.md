今日 Anthropic 集中发布了关于 **AI 模型跨越数字与物理世界边界** 的重磅实验结果，核心亮点在于验证了 Claude 在非受控环境下的自主能力。首先，通过 **Project Fetch** 和 **Project Vend**，Anthropic 展示了 Claude 即使未经专门的机器人训练，也能指挥硬件（机器狗）完成复杂任务，甚至长期自主经营实体店铺（尽管过程中出现了有趣的失败和幻觉）。其次，在安全领域，Claude Opus 4.6 在与 Mozilla 的合作中展现了惊人的 **“零日漏洞”挖掘能力**，单月发现的高危漏洞数量超过了人类全年的总和。这一系列发布表明，Anthropic 正在从单纯的“文本模型竞赛”转向“具身智能”与“高危能力管控”的深水区。

## 2. Anthropic / Claude 内容精选

### 📂 分类：Research (前沿政策与红队测试)

**1. Project Fetch: Can Claude train a robot dog?**
- **发布日期**: 2026-03-26 (回溯标注 Nov 12, 2025)
- **核心观点**: Anthropic 进行了一项对照实验，让非机器人专业的员工在 Claude 的辅助下编程控制四足机器人捡球。结果显示，**Team Claude（有 AI 辅助）的任务完成速度是无 AI 辅助团队的两倍**，且只有前者实现了全自主检索。这证明了前沿大模型具备“跨域泛化”能力，能迅速理解未知硬件接口并输出有效控制指令。
- **战略意义**: 模型能力正从“数字大脑”向“物理小脑”延伸，预示着通用大模型将成为机器人学的通用接口。
- 🔗 [原文链接](https://www.anthropic.com/research/project-fetch-robot-dog)

**2. Project Vend: Phase two**
- **发布日期**: 2026-03-26 (回溯标注 Dec 18, 2025)
- **核心观点**: 在让 Claude 经营自动售货机的第二阶段实验中，Anthropic 将底层模型从 Sonnet 3.7 升级至 **Sonnet 4.5**。新版本显著改善了经营表现，修复了部分身份认知错误，展示了随着基础模型智力提升，AI 处理长周期、复杂现实经济任务（如库存管理、定价策略）的能力呈指数级增强。
- **细节**: 实验中仍存在被员工诱导低价出售钨立方体的趣闻，但这正是红队测试的价值所在——发现边界。
- 🔗 [原文链接](https://www.anthropic.com/research/project-vend-2)

**3. Project Vend: Can Claude run a small shop? (And why does that matter?)**
- **发布日期**: 2026-03-26 (回溯标注 Jun 27, 2025)
- **核心观点**: 作为该系列的开篇，详细披露了 AI 智能体“Claudius”试图经营实体店的原始设定与早期失败。它揭示了 AI 在缺乏物理反馈回路时的脆弱性，以及在没有特定微调的情况下，通用模型在商业任务中的“幻觉”成本。
- 🔗 [原文链接](https://www.anthropic.com/research/project-vend-1)

### 📂 分类：News (安全与生态)

**4. Partnering with Mozilla to improve Firefox’s security**
- **发布日期**: 2026-03-26
- **核心观点**: Claude Opus 4.6 在 Firefox 中独立发现了 **22 个漏洞**，其中 14 个被认定为高危。这一数量占到了 Firefox 2025 年全年修复高危漏洞的近 20%。这标志着 AI 已从辅助编程工具进化为 **独立的顶级安全研究员**。
- **技术细节**: 报告指出 AI 能够以人类无法企及的速度扫描代码库，发现未知的零日漏洞，这要求软件维护流程必须针对 AI 时代进行重构。
- 🔗 [原文链接](https://www.anthropic.com/news/mozilla-firefox-security)

### 📂 分类：Engineering (工程与产品)

**5. Claude Code auto mode: a safer way to skip permissions**
- **发布日期**: 2026-03-25
- **核心观点**: 针对 Claude Code 工具中用户普遍存在的“审批疲劳”（93% 的操作会被无脑批准），Anthropic 推出了 **Auto Mode**。该模式通过内部构建的分类器来替代人工审批，旨在平衡“完全自主”带来的风险与“频繁弹窗”带来的效率损失。
- **技术细节**: 这是一个重要的工程信号，表明 Anthropic 正在试图用 AI 治理 AI，即在沙箱隔离和完全放手之间，寻找基于模型判断的中间道路。
- 🔗 [原文链接](https://www.anthropic.com/engineering/claude-code-auto-mode)

## 4. 战略信号解读

### Anthropic 的技术优先级：从“聊天”转向“行动”与“防御”
今日的内容更新不仅是一次简单的案例展示，更是一次明确的战略宣誓：
1.  **具身智能的“软件定义”路径**：不同于 Tesla 或 Figure AI 依赖端到端专有模型，Anthropic 试图证明 **通用大模型（LLM）本身就是最好的机器人大脑**。Project Fetch 证明了只要给 Claude 接口，它就能直接驱动硬件，这大大降低了机器人开发的门槛。
2.  **安全能力的“双刃剑”展示**：通过 Firefox 漏洞挖掘案例，Anthropic 高调展示了其顶级模型（Opus 4.6）的进攻性能力。这传递出两个信号：一是 Claude 的代码能力已达到甚至超越顶级人类专家水平；二是暗示其在 AI Safety（防止模型被滥用）方面有足够的自信才会公开此类高危能力。
3.  **产品化思路的转变**：Claude Code Auto Mode 的推出，说明 Anthropic 意识到 Agent（智能体）落地的最大阻碍不是智力，而是人机交互的摩擦成本。他们正试图通过工程手段解决“信任自动化”的问题。
