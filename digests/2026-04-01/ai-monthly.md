## 1. 月度要闻

本月行业经历了剧烈的分化与重组，视频生成赛道遭遇滑铁卢，而代码智能体赛道则迎来爆发式增长。

*   **03-26 | OpenAI 关闭 Sora 平台，视频生成赛道遇冷**
    上线仅 6 个月即停服，迪士尼撤回 10 亿美元投资。这一事件标志着生成式 AI 在视频领域的“算力经济性”面临崩盘，高成本低收益的模式在商业化验证中失败。
*   **03-26 | Anthropic “实体 Agent” 密集发布**
    一周内连发 8 篇文章，展示 Claude 自主经营实体商店 (`Project Vend`)、控制机器狗及在 Firefox 中单月发现 22 个 0-day 漏洞。这宣告了 AI 从“对话者”向“具身智能”与“超级安全员”的跨越。
*   **03-20 | OpenAI 收购 Astral，收拢底层工具链**
    收购 Ruff 和 uv 的开发团队 Astral，意在补强底层 Rust 工具链，为构建 AI 原生“超级应用”和自主编程智能体铺路。
*   **03-22 | Anthropic 发布 Claude 4.5/4.6 及企业级平台**
    确认推出 Opus 4.6，并全面开放 100 万 Token 上下文。同时发布 **Prompt Caching**、**Admin API** 及 **Projects/Skills**，标志着其从单一模型商向企业级 Agent 平台的转型。
*   **03-27 | 字节跳动开源 `deer-flow` SuperAgent**
    推出集沙箱、记忆、子代理于一体的长周期 Agent 框架，处理耗时数小时的复杂任务。这标志着大厂开始发力复杂的 Agent 编排“操作系统”。
*   **03-18 | GPT-5 系列矩阵曝光与 Codex Agent Loop 发布**
    OpenAI 官网泄露 GPT-5.2/5.3/5.4 完整矩阵，并正式发布 **Codex Agent Loop** 架构，代码生成正式从“补全”进化为“自主循环执行”。
*   **03-30 | Claude Code 生态爆发与“Mythos”泄露传闻**
    GitHub Trending 被 Claude 相关工具霸榜，围绕 Claude Code 的全栈工具链形成。月底关于 "Claude Mythos" 新模型及内部文件泄露的传闻引发社区震动。

## 2. CLI 工具月度进展

本月 CLI 工具全面向 **“自主代理”** 进化，核心矛盾集中在 **Token 成本透明度**、**跨平台稳定性**（Windows/WSL）和 **企业级安全控制**。

### 核心动态
*   **Claude Code (生态霸主)**:
    *   **进展**: 发布 v2.1.80 系列，强化了 Hook 系统和权限控制。生态爆发式增长，从监控 (`claude-hud`) 到多智能体编排 (`oh-my-claudecode`) 的全栈工具链形成。
    *   **痛点**: **Max 订阅配额“秒没”** 和 **Token 消耗黑盒** 引发广泛焦虑。严重的安全事故（自主清库）和长上下文导致的“智能下降”成为最大槽点。
*   **OpenAI Codex (激进重构)**:
    *   **进展**: 正在进行激进的 **Rust 底层架构重构**，从 TUI 向 `App Server` 模式迁移，引入独立的 `codex-exec-server` 支持远程/异步执行。
    *   **痛点**: **计费异常与 Token“燃尽”** 问题严重，导致用户信任危机；Intel Mac 支持缺失。
*   **Gemini CLI & Qwen Code (工程追赶)**:
    *   **动态**: 侧重于工程化架构创新（SDD 工作流、AST 感知）。**Windows 平台体验**（Shell 失效、权限弹窗）是共同的修复重点。Qwen Code 发布了 `v0.13.1-nightly`，优化了本地模型集成。

### 行业共性
*   **AGENTS.md 标准化**: 跨工具配置共享成为社区强烈诉求 (3000+ 👍)。
*   **Hooks 系统**: 已成为标配，用于实现 CI/CD 集成和自定义审批流。
*   **资源管理**: 内存泄漏 和 Token 消耗失控是全行业的共性技术债。

## 3. AI Agent 生态月报

### 格局变化
本月 Agent 生态从“通用聊天”全面转向 **“垂直技能化”** 和 **“多智能体协作”**。

*   **OpenClaw (中坚力量)**:
    *   **状态**: 处于高强度迭代期（日均 PR/Issue 500+），但也经历了 v2026.3.12 的“稳定性危机”（OOM 崩溃、UI 回归）。
    *   **架构**: 移除过时 OAuth，引入企业级密钥管理，修复了 WebSocket 劫持等高危漏洞。社区对 **Linux/Windows 原生客户端** 及 **飞书/钉钉集成** 的呼声极高。
*   **新兴势力**:
    *   **Agency-Agents**: 提出了完整的“AI 代理机构”方案，Agent 从单点工具向“虚拟团队”演进。
    *   **OpenViking (字节)**: 用文件系统范式管理 Agent 上下文，解决长周期任务痛点。
    *   **InsForge**: 专注于 Agent 的后端基础设施（DB/API/部署）。

### 值得关注的信号
*   **Agentic UX**: 开发者对 Agent 的可视化、控制权和可观测性需求激增，相关工具在 GitHub 上霸榜。
*   **MCP 协议**: 已成为连接外部工具的事实标准，但在鉴权稳定性和连接超时上仍有大量 Bug。

## 4. RL 开源生态月报

本月 RL 生态的核心关键词是 **“多模态 (VLM)”**、**“异构算力”** 与 **“算法融合”**。

*   **框架竞争 (verl, TRL, slime, OpenRLHF)**:
    *   **verl**: 活跃度极高，正向 **Diffusion RL**（图像/视频生成）和 **NPU/GB200** 异构硬件拓展。
    *   **TRL**: 正在重构底层架构，引入 `PSPO` 和 `DGPO` 等新算法，重点解决 VLM 训练的显存瓶颈。
    *   **OpenRLHF**: 开始移除 DeepSpeed 依赖，全面拥抱 PyTorch 原生 FSDP2，标志着底层基础设施的重大迁移。
    *   **Slime**: 集中攻坚 Qwen3-VL 等多模态模型在分布式训练下的死锁和显存溢出问题。
*   **算法趋势**: 社区正在重新审视 **GRPO 与 PPO 的融合**，试图在 GRPO 的显存优势与 PPO 的稳定性之间寻找平衡（如引入 Value Model）。
*   **基础设施**: **vLLM 0.10+** 的深度集成成为标配，用于解决生成速度瓶颈；同时 **FP8 训练** 和 **权重同步机制** 是工程优化的焦点。

## 5. 技术趋势总结

1.  **Agent OS 的崛起**: AI 工具正从单一插件演变为具备沙箱、内存、文件系统和权限管理的“操作系统”。
2.  **Rust 化浪潮**: 为了应对 Agent 长时间运行的高性能和内存安全需求，底层工具链正加速从 Python/Go 迁移。
3.  **成本焦虑催生精细化运营**: “Token 燃烧”引发了开发者的普遍恐慌，实时监控、上下文压缩和 Prompt Caching 成为刚需。
4.  **视频生成的经济性瓶颈**: Sora 的失败证明，在算力成本未数量级下降前，To C 的高成本生成视频模式难以跑通。

## 8. 下月展望

基于 3 月的趋势，我们对 4 月做出如下预判：
