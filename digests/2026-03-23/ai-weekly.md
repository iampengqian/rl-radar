### 2. CLI 工具进展：架构重构与安全边界

本周 AI CLI 工具从“辅助对话”全面转向 **“自主执行”**，核心竞争点聚焦在 **安全沙箱**、**上下文压缩** 和 **IDE 深度集成**。

- **Claude Code (Anthropic)**:
  - **生态爆发**：发布了 v2.1.80 系列版本，强化了 Hook 系统和权限控制。
  - **痛点**：社区爆发了对 **安全红线** 和 **数据丢失** 的焦虑（如自主清库事故），以及长上下文带来的 Token 消耗恐慌。
  - **趋势**：正从单一工具演变为具备插件系统的 **"Agent OS"**。

- **OpenAI Codex**:
  - **底层重构**：正在进行激进的 **Rust 架构迁移**，引入独立的 `codex-exec-server` 以支持远程/异步执行。
  - **安全**：重构权限系统（`guardian_approval`），试图在底层解决 Agent 越权问题。
  - **集成**：推进 VS Code 插件深度集成，但被指 Token 消耗异常。

- **Gemini CLI**:
  - **智能化**：引入 AST 感知文件读取和 Memory 子系统，试图减少 Agent 的幻觉。
  - **架构**：向深度解耦的微服务架构演进，重点解决 Plan Mode 的稳定性。

- **Qwen Code**:
  - **工程化**：发布 v0.13.0，率先实现只读工具并发执行，引入动态截断和输出掩码架构。
  - **短板**：VS Code 插件连接稳定性仍是主要投诉点。

- **共同趋势**：**并发执行**、**长上下文管理**（动态压缩）以及 **权限系统的精细化** 成为全行业共性需求。

### 4. RL 开源生态：异步与异构的军备竞赛

本周 RL 生态的核心战场转移到了 **LLM Post-training** 的工程深水区，**verl**、**TRL** 和 **OpenRLHF** 三足鼎立。

- **架构演进**：
  - **异步化**：**TRL** 推进 Async GRPO，**AReaL** 探索异步架构，试图解耦 Rollout 生成与参数更新，以突破训练吞吐瓶颈。
  - **去 DeepSpeed 化**：**OpenRLHF** 和 **verl** 均在全面拥抱 PyTorch FSDP2，降低维护复杂度并提升显存效率。

- **算力适配**：
  - **异构计算**：**verl** 和 **ROLL** 均在加速适配华为昇腾（NPU）和国产 CoreX 芯片，显示出 RL 训练对硬件生态的争夺。
  - **长上下文**：**Open Instruct** 和 **Slime** 重点攻克 32k+ 长序列训练中的通信瓶颈和显存溢出（OOM）问题。

- **算法创新**：
  - **自蒸馏（Self-Distillation）**：**TRL** 引入 SDPO，**verl** 引入 On-Policy Distillation，利用模型自身高质量轨迹进行优化成为新趋势。
