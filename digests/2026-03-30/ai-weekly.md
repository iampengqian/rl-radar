## 2. CLI 工具进展

本周 AI CLI 工具生态从“辅助编程”全面迈向**自主代理**，核心矛盾集中在**Token 成本透明度**、**跨平台稳定性**（Windows/WSL）和**企业级安全控制**。

*   **Claude Code**:
    *   **核心动态**: 处于舆论与技术的前沿。虽然社区认可其代码能力，但 **Max 订阅配额“秒没”** 和 **Token 消耗黑盒** 引发广泛焦虑。官方推出了 **Auto Mode** 试图平衡自动化与权限弹窗的疲劳问题。
    *   **技术痛点**: 严重的安全事故（自主清库）和长上下文导致的“智能下降”成为本周最大槽点。
*   **OpenAI Codex**:
    *   **核心动态**: 正在进行激进的 **Rust 底层架构重构**，试图解决沙箱稳定性和遥测问题。
    *   **争议**: 用户报告 **Token“燃尽”** 问题严重，后台静默消耗引发信任危机。
*   **Gemini CLI & Qwen Code**:
    *   **动态**: 侧重于工程化架构创新（SDD 工作流、AST 感知）。**Windows 平台体验**（Shell 失效、权限弹窗）是共同的修复重点。Qwen Code 发布了 `v0.13.1-nightly`，优化了本地模型集成。
*   **行业共性**:
    *   **Hooks 系统**: 已成为标配，用于实现 CI/CD 集成和自定义审批流。
    *   **MCP 协议**: 各家均在加速集成，但在鉴权稳定性和连接超时上仍有大量 Bug。

## 3. AI Agent 生态

**OpenClaw** 本周处于高强度迭代期，日更 PR/Issue 均达 500 条上限，显示出项目正处于成熟化的关键节点。

*   **架构重构**: 重点修复了网关重启后的会话卡死问题，并引入了 `FlushGuard` 和 `SessionPruner` 以增强持久化和自动清理能力。
*   **生态集成**: 移除了过时的 Qwen OAuth，转向标准的 Model Studio；大幅修复了 Slack、Telegram 和 WhatsApp 的连接稳定性。
*   **社区呼声**: **Linux/Windows 原生客户端** 需求居高不下（Issue #75），同时 Docker 环境下依赖安装困难成为新的部署痛点。
*   **赛道趋势**: 开源 Agent 正从通用聊天转向**垂直技能化**。具备自主网络调研能力的 `last30days-skill` 和软件开发方法论 `superpowers` 成为 GitHub 新宠。

## 4. RL 开源生态

本周 RL 生态的核心关键词是 **“多模态 (VLM)”**、**“异构算力”** 与 **“算法融合”**。

*   **框架竞争 (verl, TRL, slime, OpenRLHF)**:
    *   **verl**: 活跃度极高，正向 **Diffusion RL**（图像/视频生成）和 **NPU/GB200** 异构硬件拓展。
    *   **TRL**: 正在重构底层架构，引入 `PSPO` 和 `DGPO` 等新算法，重点解决 VLM 训练的显存瓶颈。
    *   **OpenRLHF**: 开始移除 DeepSpeed 依赖，全面拥抱 PyTorch 原生 FSDP2，标志着底层基础设施的重大迁移。
    *   **Slime**: 集中攻坚 Qwen3-VL 等多模态模型在分布式训练下的死锁和显存溢出问题。
*   **算法趋势**: 社区正在重新审视 **GRPO 与 PPO 的融合**，试图在 GRPO 的显存优势与 PPO 的稳定性之间寻找平衡点（如引入 Value Model）。
*   **基础设施**: **vLLM 0.10+** 的深度集成成为标配，用于解决生成速度瓶颈；同时 **FP8 训练** 和 **权重同步机制** 是工程优化的焦点。

## 5. 开源趋势

*   **Claude Code 全家桶**: 本周 GitHub 的绝对主角。`everything-claude-code` 和 `learn-claude-code` 暴涨，开发者正围绕 Claude 构建从底层架构到 UI 优化的完整工具链。
*   **SuperAgent 框架**: 以字节 `deer-flow` 为代表，强调**长周期任务**（数小时级）处理能力，集成了沙箱、记忆和工具调用，试图构建“Agent OS”。
*   **感知与多模态**: 
    *   **非视觉感知**: `RuView` 利用 WiFi 信号进行姿态估计，为 Agent 提供了无需摄像头的环境感知能力。
    *   **文档理解**: `Chandra` OCR 模型爆火，解决了 RAG 系统处理复杂表格和手写体的痛点。
    *   **语音交互**: 微软开源 `VibeVoice`，代表了下一代 TTS 前沿方向。
